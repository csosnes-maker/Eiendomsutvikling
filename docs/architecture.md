# Arkitektur

Dashboardet i `eiendomsutvikling` skal være ett overordnet lag.

Flyt:
1. Dashboardet henter data fra repo A
2. Dashboardet henter status fra repo B
3. Dashboardet viser alt samlet i ett grensesnitt

Første fase:
- kun lesing av data

Andre fase:
- mulighet for handlinger tilbake til utvalgte repoer
